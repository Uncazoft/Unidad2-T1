Referencias y Copias

Conceptos Clave

Objetos en JavaScript:
Son colecciones de propiedades, donde cada propiedad tiene una clave (nombre) y un valor. Los objetos pueden contener cualquier tipo de dato, incluidos otros objetos y funciones.

Copia Superficial:
Al hacer una copia superficial, como con el operador de propagacion, solo se copian las propiedades del primer nivel. Las propiedades que son objetos anidados se copian por referencia. Cambios en estos objetos anidados afectaran a ambos objetos copiados.

Mutabilidad:
Los objetos en JavaScript son mutables, lo que significa que pueden ser modificados despues de su creacion. Esto permite cambiar sus propiedades en cualquier momento.

Operador de Propagacion (...):
Este operador se usa para "expandir" un objeto en un nuevo contexto. Es especialmente util para crear copias de objetos y para combinar objetos.

Referencias:
Cuando asignas un objeto a otra variable sin hacer una copia, ambas variables referiran al mismo objeto en memoria. Modificaciones a traves de una variable afectaran a la otra.

Funciones de Cada Elemento

let: Palabra clave para declarar variables en JavaScript. Su alcance es de bloque, lo que significa que solo esta disponible dentro del bloque donde se define.

{}: Sintaxis para definir objetos literales. Dentro de las llaves, se pueden incluir propiedades y metodos.

... (Operador de Propagacion): Se usa para crear copias superficiales de objetos, expandir arrays o combinar objetos.

console.log(): Metodo del objeto console que imprime informacion en la consola, util para depuracion.

y como evitarlo:
El comportamiento se produce porque a y b son referencias al mismo objeto. Para evitar esto, puedes hacer una copia superficial del objeto usando el operador de propagacion (...) de modo que b apunte a un nuevo objeto y no al mismo que a.
